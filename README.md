# AMR-Bench-mini

AMR-Bench-mini is a focused pilot benchmark for agentic antimicrobial-resistance
reasoning, accepted as a poster paper at the FM4LS workshop at ICML 2026. It is
intentionally scoped to retrospective evidence synthesis, not clinical decision
support and not antibiotic design.

The current pilot uses public *Klebsiella pneumoniae* isolate metadata, genome
assemblies, laboratory AST records, and precomputed AMR annotation outputs.

## Paper

The camera-ready FM4LS @ ICML 2026 workshop paper is in `paper/`:

- `paper/paper.pdf`: compiled camera-ready PDF.
- `paper/paper.tex`: source for the paper.
- `paper/README.md`: paper build notes and source mapping for reported numbers.

## Repository Map

- `data/`: frozen public pilot inputs and generated task JSONL files.
- `outputs/`: processed annotator outputs and provenance snapshots used by the
  task builder.
- `results/`: baseline, model-run, audit, and paper-table artifacts.
- `scripts/`: task construction, scoring, audit, table-building, and evaluation
  entry points.
- `src/amr_bench/`: library code used by the scripts and tests.
- `tests/`: unit tests for task construction, scoring, and utility behavior.

## Tracks

The benchmark targets **multi-scale biological reasoning** about antimicrobial
resistance. The three tracks form a progression: gateway phenotype prediction →
inter-tool reconciliation → multi-scale mechanistic explanation.

### GenoPheno (gateway)

Given an isolate, antibiotic, genome metadata, and visible annotation evidence,
predict the retrospective AST phenotype and provide evidence-aware uncertainty.

### DBReconcile

Given AMRFinderPlus and ResFinder evidence for the same isolate and target
gene/family, classify the disagreement type and produce a calibrated
reconciliation.

### MechReason (headline track)

Given an isolate, antibiotic, AST phenotype, and visible annotation evidence,
produce a five-layer mechanistic explanation:

  L1 Genome    — gene/variant, location (chromosome vs plasmid), MGE context
  L2 Protein   — protein identity, native function, structural class, variant impact
  L3 Mechanism — catalytic, binding, or efflux mechanism; substrate specificity
  L4 Cell      — cellular compartment, pathway interactions, regulators upstream
  L5 Phenotype — quantitative chain from molecular event to observed MIC

Plus classify the dominant `mechanism_class` from a fixed vocabulary:
`enzymatic_inactivation, target_modification, target_protection, efflux,
permeability_loss, regulator_loss_of_function, metabolic_bypass, intrinsic,
insufficient_evidence`.

Cases where visible annotator evidence does not deterministically explain the
phenotype intentionally land on `insufficient_evidence`; the agent is expected
to recognize the gap and propose a falsifiable biological hypothesis with a
concrete validation step (HypothesisGen flavor).

## Current Data

- 60 *Klebsiella pneumoniae* isolates from BV-BRC.
- 60 local FASTA assemblies in `data/fasta/`.
- 1,410 laboratory AST records in `data/ast.json`.
- 1,201 GenoPheno + 987 DBReconcile + 924 MechReason tasks in `data/tasks/`.
- AMRFinderPlus TSV outputs in `outputs/amrfinder/` (versions pinned in `outputs/provenance.json`).
- ResFinder outputs in `outputs/resfinder/`.
- CARD data snapshot in `data/card/`.

The current data is a pilot scaffold. Do not describe it as a general AMR
benchmark across species.

## Quickstart

From this directory:

```bash
python3 -m pip install -e .
python3 scripts/build_tasks.py
python3 scripts/run_baselines.py
python3 scripts/make_manual_audit.py
PYTHONPATH=src python3 -m unittest discover -s tests
```

Generated artifacts:

```text
data/tasks/genopheno.jsonl
data/tasks/dbreconcile.jsonl
outputs/task_bundles.json
outputs/dataset_summary.json
results/rule_baseline.jsonl
results/rule_baseline_summary.json
results/rule_baseline_summary.md
results/manual_audit.tsv
```

To rebuild the paper PDF:

```bash
cd paper
latexmk -pdf -interaction=nonstopmode paper.tex
```

## Safety Scope

This benchmark evaluates retrospective AMR evidence synthesis. It excludes:

- patient treatment recommendation
- wet-lab protocol generation
- pathogen engineering
- resistance optimization
- deployable antibiotic design

Outputs that make clinical recommendations should be treated as benchmark
failures.

## Reproducibility Notes

The task builder reads existing local artifacts and does not rerun external AMR
tools. Tool-output provenance is retained in each task where available.

External tool versions and image digests are pinned in
`outputs/provenance.json` (regenerated by `scripts/capture_annotator_provenance.py`,
also auto-invoked from `scripts/run_annotators.sh`). ResFinder additionally embeds
its full software/database/command provenance in each
`outputs/resfinder/<genome>/573.json`. AMRFinderPlus TSV outputs do not embed
version info; the provenance JSON is the authoritative pin for those files.

Per-run LLM cost estimates land in each `results/llm_eval/<run>/summary.json`
under `cost_estimate_usd`, computed from `src/amr_bench/pricing.py`. Provider
prices that the local table cannot resolve are flagged with a `note`; supply
prices via `AMR_BENCH_PRICING_OVERRIDES=path/to/prices.json` to attach a
local list price without editing the table.

The published model calls used provider defaults with a shared output-token cap;
explicit reasoning-effort or thinking-budget sweeps were not run. Provider
reported reasoning/thought-token buckets are logged when exposed by the API.
